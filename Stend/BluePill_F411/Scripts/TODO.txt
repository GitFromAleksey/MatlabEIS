# Требования к скрипту-парсеру
- формат имены файла лога 
  Формат названия папки с логами: НазваниеЭксперимента_ДатаВремя, пример: "Эксперимент_13.20.15_20.10.2025"
- имя файла лога дожно сдержать частоту в герцах для парсера
  Формат названия файла лога (частота_в_герцах.log),              пример: "1000_Hz.log"

- формат одной записи строки файла лога
  {"TIME_STAMP":298,"Ch0":2598,"Ch1":1699}

- парсер должен парсить файлы логов разных частот и сохранять их в отдельный json файл для дальнейших рассчётов

# Трабования к анализатору данных
- скипт анализатора данных должен поддерживать формат выходного файла парсера
- производить рассчёт фурье для каждой частоты
- выделять из фурье результат для каждой частоты (real, imag, angle, abs) и записывать в отдельную структуру данных и сохранение в файл

- строить график фурье для каждой частоты, 
- график импеданса для параметров (real, imag, angle, abs) от частоты
- график импеданса для параметров -imag от real
